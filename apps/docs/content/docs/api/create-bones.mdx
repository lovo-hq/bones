---
title: createBones
description: The core function that powers skeleton loading states.
---

```ts
import { createBones, minMax } from "bones";

const { bone, data, repeat, lines } = createBones(dataOrPromise, options?);
```

`createBones` accepts data that may be a resolved value, a `Promise`, `null`, `undefined`, or the `forceBones` sentinel. An optional second argument controls behavior.

When only options are needed (no data to resolve), pass them as the sole argument:

```tsx
const { bone, repeat } = createBones({ loading: !moveDetails });
```

## Options

| Option    | Type      | Default | Description                                                                                                                                      |
| --------- | --------- | ------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `loading` | `boolean` | —       | When `true`, forces skeleton mode regardless of the data state. Useful for client-side data fetching with hooks that expose an `isLoading` flag. |

```tsx
const { data, isLoading } = useSWR("/api/user", fetcher);
const { bone, data: user } = createBones(data, { loading: isLoading });
```

## Return value

`createBones` returns four things:

## `bone(type, options?)`

Returns HTML attributes that turn an element into a skeleton when loading, or an empty object when data is resolved. Spread the result onto your element.

```tsx
<h2 {...bone("text", { length: 12 })}>{data?.name}</h2>
```

### Parameters

| Parameter | Type                               | Description                                       |
| --------- | ---------------------------------- | ------------------------------------------------- |
| `type`    | `"text" \| "block" \| "container"` | The skeleton visual style                         |
| `options` | `BoneOptions`                      | Optional configuration (only accepted for `text`) |

### BoneOptions

These options only apply to `"text"` bones. They are ignored for `"block"` and `"container"` types.

| Option      | Type               | Default | Description                                                                         |
| ----------- | ------------------ | ------- | ----------------------------------------------------------------------------------- |
| `length`    | `number \| MinMax` | —       | Width in `ch` units. Pass a `minMax(min, max)` value for variable-length skeletons. |
| `contained` | `boolean`          | `false` | Use inline-flex sizing instead of full-width                                        |

### `minMax(min, max)`

Creates a variable-length descriptor for the `length` option. Each `bone()` call within a `createBones` instance resolves to a different deterministic width between `min` and `max` (inclusive). This produces natural-looking skeleton lists when used inside `repeat()` or `lines()`.

```tsx
import { createBones, minMax } from "bones";

const { bone, data, repeat } = createBones(pokemon);

{
  repeat(data?.moves, 6, (item, i) => (
    <span key={item ?? i} {...bone("text", { length: minMax(4, 12), contained: true })}>
      {item}
    </span>
  ));
}
```

| Parameter | Type     | Description                 |
| --------- | -------- | --------------------------- |
| `min`     | `number` | Minimum width in `ch` units |
| `max`     | `number` | Maximum width in `ch` units |

### Bone types

**`"text"`** — for text content like headings, paragraphs, and labels. Renders a bar at the text baseline. Respects `length` and `contained` options. For multi-line text, use the `lines()` helper.

**`"block"`** — for replaced elements like images and videos. Sets a solid background color and injects a transparent pixel as `src`.

**`"container"`** — for wrapper elements with complex children. Overlays a solid background and hides children with `visibility: hidden`.

## `data`

The resolved value of the input. `undefined` while loading.

```tsx
const { bone, data } = createBones(userPromise);
// data is User | undefined
```

## `repeat(array, count, render)`

Renders a list with skeleton placeholders during loading, or the real data when resolved. The render callback receives each item (or `undefined` during loading) and its index.

```tsx
const { bone, data, repeat } = createBones(pokemon);

{
  repeat(data?.types, 2, (item, i) => (
    <TypeBadge key={item ?? i} type={item} {...bone("text", { contained: true, length: 7 })} />
  ));
}
```

During loading, `repeat` calls the render callback `count` times with `(undefined, index)`. When data is resolved, it maps over the real array — the `count` argument is ignored.

A count-only signature is also available when there is no array data to pass through:

```tsx
{
  repeat(3, (_, i) => <div key={i} {...bone("text", { length: minMax(10, 20) })} />);
}
```

## `lines(value, count, render)`

Renders multi-line paragraph skeletons during loading, or the real content when loaded. The render callback provides the wrapper element — `lines` handles the skeleton content internally.

```tsx
const { data, lines } = createBones(species);

{
  lines(data?.description, 4, (item) => <p className={styles.description}>{item}</p>);
}
```

During loading, `item` is an array of skeleton `<span>` elements — full-width lines with a shorter last line. The render callback wraps them in your element, producing a single paragraph with skeleton lines inside. Each line has its own `::after` pseudo-element with proper shimmer animation.

When data is resolved, `item` is the real value (e.g. a string). The render callback wraps it the same way, producing a single paragraph with real text. No `bone()` call needed in the callback.

## Type signature

```ts
function createBones(options: CreateBonesOptions): CreateBonesReturn<never>;
function createBones<T>(
  data: T | Promise<T> | undefined | null,
  options?: CreateBonesOptions,
): CreateBonesReturn<T>;

function minMax(min: number, max: number): MinMax;

interface MinMax {
  readonly min: number;
  readonly max: number;
}

interface CreateBonesOptions {
  loading?: boolean;
}

interface BoneOptions {
  length?: number | MinMax;
  contained?: boolean;
}

interface CreateBonesReturn<T> {
  bone: {
    (type: "text", options?: BoneOptions): Record<string, unknown>;
    (type: "block" | "container"): Record<string, unknown>;
  };
  data: T | null | undefined;
  repeat: {
    (count: number, render: (item: undefined, index: number) => ReactNode): ReactNode[];
    <U>(
      arr: U[] | undefined | null,
      count: number,
      render: (item: U | undefined, index: number) => ReactNode,
    ): ReactNode[];
  };
  lines: <V>(
    value: V | null | undefined,
    count: number,
    render: (item: V | ReactNode) => ReactNode,
  ) => ReactNode[];
}
```
