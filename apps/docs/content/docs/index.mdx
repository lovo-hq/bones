Bones lets you write a single component that handles both its loaded and loading states. Instead of building separate skeleton components, you add a few attributes to your existing markup and Bones takes care of the rest.

## How it works

`createBones` accepts data that may or may not be resolved yet. When the data is still loading, its `bone` function returns HTML attributes that turn your elements into skeletons. When the data arrives, `bone` returns an empty object and your component renders normally.

function UserCard({ user }: { user: Promise<User> }) {
  const { bone, data, lines } = createBones(user);

    <div style={{ display: "flex", gap: 12, alignItems: "flex-start" }}>
      <img
        src={data?.avatar}
        alt=""

        style={{ borderRadius: "50%" }}
        {...bone("block")}
      />

        {lines(data?.bio, 2, (item) => (

## Key ideas

- **Same component, both states** — one component definition, no separate skeleton.
- **No hooks** — `createBones` is a plain function, not a hook.
- **Promise-native** — works with React Suspense out of the box via `readPromise`.
- **CSS-driven** — all skeleton visuals are pure CSS. Customize with CSS custom properties.
- **Accessible** — sets `aria-busy="true"` on loading elements automatically.

## A different approach

Most skeleton libraries require you to build and maintain separate skeleton components that mirror your real UI. Bones takes a different approach — you write one component that handles both states.

**Traditional approach**

```tsx
// UserCard.tsx
function UserCard({ user }) {
  return (
    <div>
      <img src={user.avatar} alt="" />
      <h2>{user.name}</h2>
      <p>{user.bio}</p>
    </div>
  );
}

// UserCardSkeleton.tsx
function UserCardSkeleton() {
  return (
    <div>
      <Skeleton circle width={48} height={48} />
      <Skeleton width={120} />
      <Skeleton count={2} />
    </div>
  );
}

// Page.tsx — conditional rendering
function Page() {
  const { data, isLoading } = useFetch("/api/user");

  if (isLoading) return <UserCardSkeleton />;
  return <UserCard user={data} />;
}
```

**With Bones**

```tsx
// UserCard.tsx — handles both states
function UserCard({ user }: { user: Promise<User> }) {
  const { bone, data, lines } = createBones(user);

  return (
    <div>
      <img
        src={data?.avatar}
        alt=""
        {...bone("block")}
      />
      <h2 {...bone("text", { length: 12 })}>
        {data?.name}
      </h2>
      {lines(data?.bio, 2, (item) => (
        <p>{item}</p>
      ))}
    </div>
  );
}

// Page.tsx — no conditional rendering
function Page() {
  return (
    <Bones>
      <UserCard user={fetchUser()} />
    </Bones>
  );
}
```
