## Installation

```bash
npm install bones
```

## Import the CSS

Bones uses a small CSS file for skeleton visuals. Import it once in your app's root layout or entry point:

```tsx
import "bones/css";
```

## Basic usage

Pass data (or a promise of data) to `createBones`. Spread the `bone` function's return value onto elements that should show skeletons while loading.

```tsx
import { createBones } from "bones";

function ProfileCard({ user }: { user: Promise<User> | User }) {
  const { bone, data, lines } = createBones(user);

  return (
    <div>
      <img src={data?.avatar} width={80} height={80} {...bone("block")} />
      <h3 {...bone("text", { length: 10 })}>{data?.name}</h3>
      {lines(data?.bio, 3, (item) => (
        <p>{item}</p>
      ))}
    </div>
  );
}
```

## Add a Suspense boundary

Wrap components that receive promises in `<Bones>` to automatically handle the Suspense fallback:

```tsx
import { Bones } from "bones";

export default function Page() {
  return (
    <Bones>
      <ProfileCard user={fetchUser()} />
    </Bones>
  );
}
```

When the promise is pending, `<Bones>` renders the same `<ProfileCard>` tree with skeletons visible. When the promise resolves, the real content appears.

## Bone types

| Type          | Use for                               | Example                              |
| ------------- | ------------------------------------- | ------------------------------------ |
| `"text"`      | Headings, paragraphs, labels          | `<h2 {...bone("text")}>`             |
| `"block"`     | Images, avatars, thumbnails           | `<img src={…} {...bone("block")} />` |
| `"container"` | Cards, wrappers with complex children | `<div {...bone("container")}>`       |

## Previewing skeletons during development

Use `forceBones` to see what a component's skeleton looks like without waiting for a real loading state:

```tsx
import { createBones, forceBones } from "bones";

// Pass forceBones in place of any promise prop:
<ProfileCard user={forceBones} />;
```

To force an entire subtree into skeleton mode at once, wrap it with `<BonesForce>`:

```tsx
import { BonesForce } from "bones";

<BonesForce>
  <ProfileCard />
  <PostList />
</BonesForce>;
```

Both approaches work without any Suspense boundaries or async data — just render and see the skeletons immediately. This is useful for tweaking styles, building out new components, or demoing loading states in Storybook.
