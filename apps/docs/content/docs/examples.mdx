## Suspense streaming

The most common pattern: pass a promise to a component wrapped in `<Bones>`.

```tsx
import { Bones } from "bones";

async function fetchUser(): Promise<User> {
  const res = await fetch("/api/user");
  return res.json();
}

export default function Page() {
  return (
    <Bones>
      <UserCard user={fetchUser()} />
    </Bones>
  );
}

function UserCard({ user }: { user: Promise<User> }) {
  const { bone, data, lines } = createBones(user);

  return (
    <div>
      <img src={data?.avatar} width={64} height={64} {...bone("block")} />
      <h2 {...bone("text", { length: 12 })}>{data?.name}</h2>
      {lines(data?.bio, 2, (item) => (
        <p>{item}</p>
      ))}
    </div>
  );
}
```

## Forced skeletons for previews

Use `<BonesForce>` to show skeleton states without real data:

```tsx
import { BonesForce } from "bones";

function SkeletonDemo() {
  return (
    <div style={{ display: "grid", gridTemplateColumns: "1fr 1fr", gap: 16 }}>
      {/* Loaded state */}
      <UserCard user={Promise.resolve(mockUser)} />

      {/* Skeleton state */}
      <BonesForce>
        <UserCard />
      </BonesForce>
    </div>
  );
}
```

## Client-side data fetching

When using hooks like React Query, SWR, or `useState` + `useEffect`, pass the `loading` option to control skeleton state. The options-only overload means you don't need to pass data as the first argument:

```tsx
import { createBones } from "bones";
import useSWR from "swr";

function UserCard() {
  const { data, isLoading } = useSWR("/api/user", fetcher);
  const { bone, data: user, lines } = createBones(data, { loading: isLoading });

  return (
    <div>
      <img src={user?.avatar} width={64} height={64} {...bone("block")} />
      <h2 {...bone("text", { length: 12 })}>{user?.name}</h2>
      {lines(user?.bio, 2, (item) => (
        <p>{item}</p>
      ))}
    </div>
  );
}
```

The component renders skeletons while `isLoading` is `true`, then shows content once data arrives — no Suspense boundary required.

When you only need `bone` and `repeat` helpers (no data to resolve), use the options-only signature:

```tsx
const { bone, repeat } = createBones({ loading: !moveDetails });
```

## Multi-line text with `lines()`

Use the `lines` helper for paragraphs and descriptions. It generates skeleton `<span>` elements inside your wrapper — full-width lines with a shorter last line, each with its own shimmer animation:

```tsx
import { createBones } from "bones";

function ArticleCard({ article }: { article: Promise<Article> }) {
  const { bone, data, lines } = createBones(article);

  return (
    <article>
      <h3 {...bone("text", { length: 20 })}>{data?.title}</h3>
      {lines(data?.excerpt, 3, (item) => (
        <p>{item}</p>
      ))}
      <span {...bone("text", { length: 8, contained: true })}>{data?.readTime}</span>
    </article>
  );
}
```

During loading, `item` is an array of skeleton spans that render inside the `<p>`. When data resolves, `item` is the real excerpt string. No `bone()` call needed in the callback — `lines` handles the skeleton content internally.

## List placeholders with `repeat()`

Use `repeat` to render a fixed number of placeholder items while data loads. The render callback receives each item (or `undefined` during loading) and its index:

```tsx
import { createBones, minMax } from "bones";

function TagList({ pokemon }: { pokemon: Promise<Pokemon> }) {
  const { bone, data, repeat } = createBones(pokemon);

  return (
    <div>
      <h3 {...bone("text", { length: 9 })}>{data?.name}</h3>
      <div style={{ display: "flex", gap: 4 }}>
        {repeat(data?.types, 2, (item, i) => (
          <span key={item ?? i} {...bone("text", { contained: true, length: 7 })}>
            {item}
          </span>
        ))}
      </div>
    </div>
  );
}
```

`repeat(data?.types, 2, render)` calls the render callback twice with `(undefined, index)` while loading (rendering two skeleton pills), then maps over the real `types` array once data arrives — the count is ignored when loaded.

Use `minMax` for natural-looking varied widths in skeleton lists:

```tsx
{repeat(data?.moves, 10, (item, i) => (
  <span key={item ?? i} {...bone("text", { length: minMax(5, 12), contained: true })}>
    {item}
  </span>
))}
```

A count-only signature is available when there's no array to pass through:

```tsx
{repeat(3, (_, i) => (
  <div key={i} className={styles.row} {...bone("text", { length: minMax(10, 20) })} />
))}
```

## Skeleton animations

By default, skeletons are static. Add the `data-bone-animate` attribute to any parent element to animate all skeletons inside it.

### Shimmer

A horizontal highlight sweep:

```tsx
<div data-bone-animate="shimmer">
  <Bones>
    <UserCard user={fetchUser()} />
  </Bones>
</div>
```

### Pulse

A gentle opacity fade:

```tsx
<div data-bone-animate="pulse">
  <Bones>
    <UserCard user={fetchUser()} />
  </Bones>
</div>
```

Bones automatically applies `bone-pulse` when `prefers-reduced-motion: reduce` is active, so reduced-motion users get a comfortable fallback regardless of which animation you choose.

### Global animation

Set the attribute on `<body>` to animate every skeleton in your app:

```tsx
<body data-bone-animate="shimmer">
```

Individual sections can override the global animation:

```tsx
<body data-bone-animate="shimmer">
  {/* These skeletons shimmer */}
  <UserCard user={fetchUser()} />

  {/* These skeletons pulse instead */}
  <div data-bone-animate="pulse">
    <Sidebar items={fetchItems()} />
  </div>
</body>
```
